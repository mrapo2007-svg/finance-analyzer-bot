## Problem

## Problem

Человек хочет понимать структуру своих трат, но банковская выписка — это
простыня из сотен строк. Одни не понимают, куда уходит доход; другие
в целом знают, но вручную разносить операции по категориям долго и лень.
В обоих случаях люди бросают это дело, и контроля над деньгами нет.

## User

Арман, 24, фрилансер. Доход нерегулярный, тратит с карты, выписку из банка может выгрузить в PDF, но смотреть в неё боится. Не бухгалтер, считать вручную не хочет.

## Success metric

На тестовой выписке из 30+ операций бот возвращает Excel, где ≥80% операций отнесены к верной категории.

## Features v1

### F1: Загрузка выписки и разбор транзакций

**Acceptance:** Given Арман открыл бота в Telegram, When отправляет PDF-выписку, Then бот извлекает транзакции (дата, сумма, описание), сохраняет их в `transactions` и отвечает «Нашёл N операций ✅».

### F2: Сводка по категориям в Excel

**Acceptance:** Given в базе есть разобранные транзакции, When Арман отправляет команду `/report`, Then бот относит каждую операцию к категории (еда, аренда, развлечения, переводы, прочее) и присылает файл `.xlsx` с суммой по каждой категории.

## User flows

### Flow 1: Арман разбирает выписку

1. Арман открывает бота → видит «Пришли PDF-выписку из банка».
2. Отправляет PDF.
3. Система парсит операции и пишет их в `transactions`.
4. Арман видит «Нашёл 142 операции ✅».
5. Отправляет `/report`.
6. Получает Excel: категории и суммы по каждой.

## Out of scope (v1)

- ИИ-советы («ты много тратишь на игры»)
- Разбивка трат по людям (на дочку, на друзей)
- Роль бухгалтера компании и доступ к чужим таблицам
- Рекомендации более экономных альтернатив
- Реальные банковские данные живых людей (только тестовая выписка)
- Шифрование / RLS / продвинутая защита данных
- Несколько банков и форматов выписок (берём один)
